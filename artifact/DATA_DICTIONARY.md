# TAISAP Artifact Data Dictionary

This file defines the public schema expected for the TAISAP evidence artifact. Values must be generated by scripts under `artifact/scripts/` and recorded in `artifact_manifest.json`.

## Shared Columns

| Column | Description |
|---|---|
| `source_commit` | Git commit used to generate the row. |
| `policy_version` | Policy/gate version active during generation. |
| `backend` / `backend_id` | LLM backend identifier or deterministic backend label. |
| `session_id` | De-identified session identifier. |
| `probe_id` | Stable adversarial or benign probe identifier. |
| `prompt_family` | One of the seven adversarial prompt families where applicable. |
| `prompt_template_id` | Cluster identifier for ASR bootstrap and sensitivity analyses. |
| `attack_category` | Normalized attack category for adversarial probes. |
| `language` | Prompt language or mixed-language label. |
| `wrapper_variant` | Prompt wrapper or scenario variant. |
| `configuration` | Evaluated system configuration, such as `B2`, `B2-PromptSafe`, `B2-ContextOnly`, or `B3`. |

## Authority Path Decomposition

Required file: `artifact/data/authority_path/authority_path_decomposition.csv`.

| Column | Description |
|---|---|
| `expected_route` | Route expected by policy for the probe. |
| `actual_route` | Route selected by the broker. |
| `routing_miss` | Boolean indicating that the expected and actual policy routes differ in a security-relevant way. |
| `llm_invoked` | Boolean indicating that the probe reached an LLM backend. |
| `unsafe_draft_generated` | Boolean indicating that an unsafe model draft was produced. |
| `post_filter_blocked` | Boolean indicating that the post-filter blocked the draft. |
| `svl_delta_proposed` | Boolean indicating that a state delta was proposed. |
| `svl_delta_rejected` | Boolean indicating that SVL rejected the proposed delta. |
| `svl_rollback` | Boolean indicating that the transaction rolled back. |
| `attacker_visible_violation` | Boolean indicating an attacker-visible content violation. |
| `execution_authority_expanded` | Boolean indicating execution-authority expansion. |
| `state_authority_expanded` | Boolean indicating state-authority expansion. |
| `egress_authority_expanded` | Boolean indicating egress-authority expansion. |
| `root_cause` | Unique primary root cause from the preregistered taxonomy. |

Allowed `root_cause` values are `routing_regex_miss`, `multilingual_bypass`, `token_smuggling`, `semantic_boundary_case`, `unsafe_draft_blocked`, `sanitizer_miss`, `persona_only_failure`, `invalid_delta_rejected`, `visible_policy_violation`, and `authority_expansion`.

## Parser Fuzzing

Required files: `artifact/data/fuzzing/parser_fuzz_cases.csv` and `artifact/data/fuzzing/parser_fuzz_campaigns.csv`.

`parser_fuzz_cases.csv` must include `case_id`, `campaign_id`, `seed`, `mutation_family`, `input_length`, `annotation_count`, `valid_syntax`, `parse_result`, `reject_reason`, `processing_time_ms`, `timeout`, `exception`, `candidate_delta_count`, `state_commit`, `unauthorized_commit`, `fail_open`, `coverage_hash`, `parser_version`, and `source_commit`.

`parser_fuzz_campaigns.csv` must include `campaign_id`, `start_time`, `end_time`, `case_count`, `seed`, `dictionary_version`, `parser_commit`, `machine_spec`, `python_version`, `coverage_tool`, `branch_coverage`, `line_coverage`, and `peak_memory_mb`.

## Broker Load Runs

Required file: `artifact/data/broker_load/broker_load_runs.csv`.

Each row is one independent run and must include concurrency, duration, submitted/completed commands, workload mix, seed, throughput, queue depth, latency percentiles, event-loop lag, timeout and ERRSIM fallback counts, connection and telemetry drops, CPU/RAM/GPU peaks, fail-open count, and authority-expansion count.

## Human Audit

Required file: `artifact/data/human_audit/human_audit_labels.csv`.

Each row is one reviewer label for one sample and must include `sample_id`, `sampling_stratum`, `configuration`, `backend`, `attack_category`, `reviewer_id`, `persona_break`, `unsafe_guidance`, `ica_error`, and `final_consensus`.

## Canonical Generated Table Inputs

The canonical CSV names used by the table-generation targets are:

| File | Description |
|---|---|
| `baseline_ablation_results.csv` | Canonical ablation summary for ICA/ASR baselines and B3. |
| `prompt_injection_expanded_results.csv` | Prompt-injection and residual-failure probe outcomes. |
| `model_generalization_results.csv` | Backend/model generalization and paired comparison outcomes. |
| `containment_stress_results.csv` | Containment stress-test outcomes. |
| `svl_state_poisoning_results.csv` | SVL state-poisoning outcomes. |
| `benign_false_positive_results.csv` | Benign-command false-positive outcomes. |
| `timing_sequence_results.csv` | Timing sequence, classifier, and paired delta-AUC outcomes. |
| `broker_load_results.csv` | Broker load repeated-run outcomes. |
| `parser_fuzz_results.csv` | Parser fuzz case/campaign outcome summary. |
| `errsim_ablation_results.csv` | ERRSIM-vs-hard-refusal direct ablation outcomes. |
